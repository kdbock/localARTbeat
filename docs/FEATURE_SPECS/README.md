# Feature Specs

## Purpose

This folder holds short specs for active feature work.

Every active feature spec should answer:

- what problem is being solved
- who the user is
- what packages/files are likely involved
- what acceptance criteria define done
- what tests and manual QA are required
- what is explicitly out of scope

## Rules

- one file per active feature
- keep specs short and current
- archive or delete stale specs after work is complete
- if a feature is not in this folder or `WORK_QUEUE.md`, it is not active work
